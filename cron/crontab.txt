# ============================================================================
#
# ⚠️  DANGER — `crontab <file>` REPLACES EVERY CRON JOB FOR THE USER. ⚠️
#
# Only use the destructive `crontab cron/crontab.txt` form on a server where
# you are certain no other cron jobs exist. Anywhere else, the install will
# silently wipe every unrelated entry on the user's crontab. Use the
# marker-based merge below instead.
#
# Safe install procedure (works whether or not other jobs exist, and is
# idempotent — repeated runs leave one CTA-INSIGHTS block in the live
# crontab, never duplicates):
#
#   1. Snapshot the live crontab:
#        crontab -l > /tmp/live.crontab
#   2. Strip any existing CTA-INSIGHTS block from the live crontab:
#        awk '/^# CTA-INSIGHTS-START/{skip=1} /^# CTA-INSIGHTS-END/{skip=0; next} !skip' \
#          /tmp/live.crontab > /tmp/live.stripped
#   3. Extract just the marker block (NOT the docs above) from this file:
#        awk '/^# CTA-INSIGHTS-START/,/^# CTA-INSIGHTS-END/' cron/crontab.txt \
#          > /tmp/cta-block.txt
#   4. Concat + install:
#        cat /tmp/live.stripped /tmp/cta-block.txt | crontab -
#   5. Verify line counts pre/post match for unrelated jobs.
#
# IMPORTANT: only the content between # CTA-INSIGHTS-START and
# # CTA-INSIGHTS-END below gets installed by the procedure above. The
# install header (REPO_ROOT, R=, comments) lives INSIDE the markers so a
# repeated merge replaces them in place rather than stacking.
#
# Update REPO_ROOT below to the repo's checkout path on the target server.

# CTA-INSIGHTS-START

# All jobs run via bin/cron-run.sh <log-name> <script> [args...] which
# handles cd, timestamping, and log redirection.
REPO_ROOT=/path/to/cta-insights
R=$REPO_ROOT/bin/cron-run.sh

# --- Trains ---
# train-alerts staggered to :02 offset so it never collides with train-pulse's
# */5 schedule — otherwise both see "no thread parent yet" at the same wall
# minute and post separate top-level posts instead of threading.
*/5     *  *  *  *   $R train-pulse      bin/train/pulse.js
4-59/15 *  *  *  *   $R train-bunching   bin/train/bunching.js
2-59/10 *  *  *  *   $R train-alerts     bin/train/alerts.js
7-59/15 *  *  *  *   $R train-gaps       bin/train/gaps.js
0       */2 *  *  *  $R train-speedmap   bin/train/speedmap.js
0       */3 *  *  *  $R train-snapshot   bin/train/snapshot.js
8       *  *  *  *   $R train-ghosts     bin/train/ghosts.js
# incident-roundup runs every 5 min on :04 offset, separate from pulse :02 /
# alerts :02 / gaps :07 so its detector-correlation read isn't racing the
# writers. Reads meta_signals for both train and bus near-misses and posts a
# single rollup when distinct-source signals on the same line/route within
# 30 min sum past the threshold.
4-59/5  *  *  *  *   $R incident-roundup bin/incident-roundup.js

# --- Buses ---
1-59/15 *  *  *  *   $R bus-bunching     bin/bus/bunching.js
*/10    *  *  *  *   $R bus-alerts       bin/bus/alerts.js
# bus-pulse staggered :03 offset so it never collides with bus-alerts'
# */10 schedule — same threading-safety reasoning as train-alerts vs
# train-pulse.
3-58/5  *  *  *  *   $R bus-pulse        bin/bus/pulse.js
6-59/15 *  *  *  *   $R bus-gaps         bin/bus/gaps.js
0       */2 *  *  *  $R bus-speedmap     bin/bus/speedmap.js
7       *  *  *  *   $R bus-ghosts       bin/bus/ghosts.js
# observe-buses runs */10 (not */5) to keep monthly bus-tracker calls
# under the 100k budget — 124 routes × 13 chunks × 144 ticks/day × 30 ≈ 56k.
# bunching/gaps/pulse all read the cache (maxStaleMs=11min) so observe-buses
# is effectively the only API call site for the all-routes workload.
*/10    *  *  *  *   $R observe-buses    scripts/observeBuses.js
# observe-trains polls Train Tracker every 2 min. Train Tracker shares the
# same 100k/month CTA budget as Bus Tracker, but one batched call returns all
# 8 lines and the scheduled detector crons consume far less than the bus side,
# leaving room. 30 calls/h × 24 × 30 ≈ 22k — well under budget. Densifies the
# observations table so held-train detection has enough motion samples per
# train to classify reliably.
*/2     *  *  *  *   $R observe-trains   scripts/observeTrains.js

# --- Recaps ---
20      10 *  *  0   $R bus-recap        bin/bus/recap.js   --window=week
25      10 *  *  0   $R train-recap      bin/train/recap.js --window=week
30      10 1  *  *   $R bus-recap        bin/bus/recap.js   --window=month
35      10 1  *  *   $R train-recap      bin/train/recap.js --window=month
